About Utility Classes

One of the most powerful parts of Solid, utility classes allow for rapid front-end development and page styling without writing much CSS. They also provide sensible defaults for things like spacing units and border radii. Like the rest of Solid, these classes are considered immutable and should never be directly overridden.

Additionally, most of the following utility classes take advantage of Solid's responsive prefixes. To declare a single value for all breakpoints, Solid requires the use of its xs- prefix.

Positioning

Solid includes classes for setting an element's position and hierarchy. For positioning, an element can be set to .xs-relative, .xs-absolute, .xs-fixed or .xs-static. Setting z-index and top, right, bottom, left positioning can be done by using the shorthand from the previous table.

.xs-relative .xs-t2 .xs-l2
.xs-relative
.xs-absolute .xs-t0 .xs-l6
.xs-absolute .xs-t1 .xs-l1 xs-z1
.xs-absolute .xs-t2 .xs-l2 xs-z2
